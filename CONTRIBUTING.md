# Contributing to Kujo Beatdown Beat Saber Map

Thanks for contributing! This project is collaborative and issue-driven.

## 1) Pick or claim an issue

- Use existing issues and templates in `.github/ISSUE_TEMPLATE/`.
- If taking an issue, leave a comment so work is visible.
- Keep one PR focused on one issue whenever possible.

## 2) Branch naming

Branch from `main` and use one of:

- `feat/<short-description>`
- `fix/<short-description>`
- `chore/<short-description>`

Examples:
- `feat/expert-drop-pass`
- `fix/hard-timing-offset`
- `chore/lightshow-cleanup`

## 3) File placement

Use consistent directories:

- `Hard/`
- `Expert/`
- `ExpertPlus/`
- `Lightshow/`
- `docs/`
- `references/`

If you add a new top-level folder, explain why in your PR.

## 4) Commit guidance

- Write clear commit messages.
- Keep unrelated edits out of the same PR.
- Add notes for known limitations.

## 5) Pull Request checklist

Include in PR body:

- Linked issue (`Closes #<number>` when appropriate)
- What changed
- How you tested (editor + in-game if possible)
- Any known follow-up work

Before requesting review, verify:

- [ ] Changes match issue scope
- [ ] No clearly broken/unplayable patterns introduced
- [ ] Timing sanity-checked for touched sections
- [ ] Paths and naming are consistent

## 6) Review expectations

- Be constructive and specific.
- Prefer timestamped feedback for mapping suggestions.
- If suggesting a blocker, explain impact and recommended fix.

## 7) Non-commercial policy

This is a fan project. Do not sell or monetize distributed map files.
